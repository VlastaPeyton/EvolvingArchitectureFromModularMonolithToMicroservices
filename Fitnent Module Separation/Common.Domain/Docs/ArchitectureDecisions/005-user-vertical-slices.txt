Datum:
	15.8.2025

Problem:
	Naci dobar nacin za struktuisanje aplikacije. 

Decision:
	Koristicu Vertical Slices arhitekturu, gde svaki business process ima svoj folder. Ovaj pristup eliminise tradicionalne foldere 
 poput Controllers, Entities, Commands, Queries... Umesto toga, sve potrebno za jedan process je smesteno u folder tog procesa.

Consqeuences:
	Zbog VSA, lakse je refaktorisati funkcionalnosti ako zatreba. 
	Razumevanje koda je lakse kada je related code grouped together in one folder.
	VSA je orijentisana vise ka business aspektu, nego ka tehnickim detaljima.
	Grupisanje kodova u procesu moze rezultovati duplikatima ako vise procesa imaju istu funkcionalnost.
