Datum:
	23.8.2025

Problem:
	Contracts module je najslozeniji module i on prvi pretenduje da poraste tj da jos logike mu treba + testability 
  tako da...

Decision:
	Clean architecture to Contracts module  + dobar je za testing.
	Layers: API: Http Request/Response, interfaces
			Application: Logika 
			Infrastructure: EF Core, Dapper, DbContext...
			Domain: Contract klasa...

Consequences:
	Laksa modifikacija kako aplikacija raste. 
	Testing of each layer separately.
	Use of layers allows to manipulate what is public and what is internal.
	Dodatni patterni zbog Clean architecture.
