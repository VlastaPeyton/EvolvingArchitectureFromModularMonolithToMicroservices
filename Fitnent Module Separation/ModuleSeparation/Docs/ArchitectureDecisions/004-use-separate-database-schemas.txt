Datum:
	15.8.2025

Problem:
	Efektni pristup za read/write data in modular monolith + da odrzim modularnost and scalability.

Decision:
	Koristim 1 SQL DB, gde svaki module ima svoje tabele kojima drugi moduli ne pristupaju direktno, vec preko Module1 Endpoint/Service (koji ce da pozove Module2 repository method to read db) koji samo koristi Module2 da dohvati podatke iz Module1 tabele.

Consqeuences:
	Podeliti DB u scheme(svaki module ima svoju schema) i onda svaki module ima svoje tabele u schema.
	Pristupanje podacima iz razlicitih modules moze postati complexno i mora se posebno pripaziti na inter-module communication i data sharing strategije. Inter-module communication odvija se putem In-memory Event Bus, ali kada predjemo u microservices, to moze prerasti u RabbitMQ.
