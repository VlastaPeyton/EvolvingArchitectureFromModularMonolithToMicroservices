Datum:
	25.8.2025

Problem:
	Tranzicija u microservices zahteva External Message Broker.
	Biram Azure Service Bus, RabbitMQ i Amazon SQS. Azure i Amazon retko ko zna da radi + placa se + los performance. 
  RabbitMQ svi znaju + djabe + ubija performance.

Decision:
	Retko ko zna Azure i Amazon kako se koriste + placaju se + performance im je slab. Zato biram RabbitMQ jer je djabe + znaju ga svi + performance ubija.

Consequences:
	RabbitMQ is open-source and free to use, which aligns with our budget constraints and cost goals.
	Team members are already familiar with RabbitMQ, which reduces the learning curve.
	RabbitMQ ima message routing, topis, RPC + je cross-platform
